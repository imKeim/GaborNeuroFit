# Миграция GaborNeuroFit на Vue 3

## 1. Цели

- Декларативный, реактивный UI (кнопки, статус-бар, модальные окна).
- Плавные, предсказуемые анимации без ручных таймаутов.
- Чёткое отделение UI от контроллеров и рендер-движков.
- Упрощение поддержки состояний кнопок во всех FSM-режимах.
- Полная обратная совместимость с бизнес-логикой и точностью Canvas.

## 2. Принципы миграции

- **Итеративно.** Переносим по одному компоненту, старый код продолжает работать.
- **Feature Freeze.** Логика ступеней, рендеринг Gabor/RDS/Synoptophore не трогается.
- **Типизация.** Сохраняем strict TypeScript, минимизируем `@ts-ignore`.
- **Производительность.** Pinia не вызывает лишних перерисовок, Canvas остаётся императивным.

## 3. Архитектурные решения

### 3.1 Управление состоянием

Используем **Pinia**:
- `useAppStore` – глобальные настройки (язык, модальные окна, пауза).
- `useGaborStore` – состояние Gabor-сессии (уровень, контраст, счёт, FSM-стейт).
- `useRdsStore` – аналогично для RDS.
- `useSynopStore` – аналогично для Synoptophore.

Старый `Store` будет обёрнут в эти хранилища постепенно, без удаления.

### 3.2 Компонентный стиль

**Composition API с `<script setup>`** – лучшая поддержка TS, чистая логика.

### 3.3 Canvas-компоненты

Создаём `GaborCanvas.vue`, `RdsCanvas.vue`, `SynopCanvas.vue`. Они получают `ref` на canvas, императивно вызывают движки рендеринга при изменении параметров (watch).

### 3.4 Анимации и переходы

Используем `<Transition>` и `<TransitionGroup>` для:
- Модальных окон (fade + scale).
- Теста слияния (плавное скрытие/показ фона).
- Индикаторов обратной связи (flash-success/error).

### 3.5 Таймеры и асинхронность

`AsyncResourceTracker` заменяется на `onScopeDispose` и штатные `setTimeout`/`requestAnimationFrame` внутри `watchEffect`, которые автоматически очищаются при размонтировании компонента.

### 3.6 Локализация

Переходим на **vue-i18n** – реактивное переключение без ручного обновления DOM.

### 3.7 Twemoji и шрифты

- Twemoji уже кэшируется локально через `generate-assets.ts`. Оставляем как есть, но удалим CDN-загрузку `twemoji.min.js`, если найдём локальный пакет. Пока приемлемо.
- Шрифты Google Fonts – можно оставить CDN (они кэшируются PWA), или скачать локально – решим позже.

## 5. Риски и защита

- **Синхронизация Canvas и Vue.** Всегда вызываем `nextTick` перед рисованием, если состояние могло измениться.
- **Утечки таймеров.** Используем `onScopeDispose` или `watchEffect` с очисткой вместо глобального трекера.
- **Потеря точности анимаций.** Оставляем длительность и easing без изменений, только Vue-транзишены.
- **Сложность отладки.** В dev-режиме Vue DevTools дадут прозрачность состояния.

## 6. График (гибкий)

- Этап 0–2: **1 неделя** (настройка, первые модалки, кнопки).
- Этап 3–4: **1–2 недели** (тест слияния, Canvas).
- Этап 5–6: **1 неделя** (чистка, тесты).

Если в любой момент устал — коммитим и оставляем. Ветка `dev` не мешает `main`.
